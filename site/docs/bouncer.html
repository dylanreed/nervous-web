Commands Reference

Bouncer Bot provides 17 slash commands for comprehensive moderation.

Timeout Management

/timeout

Temporarily restrict a user from sending messages or joining voice channels.

  • user (required) - The user to timeout
  • duration (required) - How long (e.g., 30m, 2h, 7d, indefinite)
  • reason (optional) - Reason for the timeout

/untimeout

Remove an active timeout from a user.

  • user (required) - The user to untimeout
  • reason (optional) - Reason for removing the timeout

/timeouts

View all currently active timeouts in the server.

Quarantine System

Quarantine removes all roles from a user and assigns a quarantine role, effectively isolating them while preserving their original roles for restoration.

/quarantine

Place a user in quarantine.

  • user (required) - The user to quarantine
  • duration (optional) - How long (e.g., 1h, 24h, indefinite)
  • reason (optional) - Reason for quarantine

/unquarantine

Remove a user from quarantine and restore their original roles.

  • user (required) - The user to unquarantine
  • reason (optional) - Reason for release

/quarantine-list

View all currently quarantined users.

Warnings & Infractions

/warn

Issue a warning to a user. Can be linked to server rules.

  • user (required) - The user to warn
  • rule (optional) - Rule number violated
  • reason (optional) - Additional notes

/reverse

Reverse a previous moderation action (warning, timeout, quarantine, etc.).

  • reason (required) - Reason for reversing
  • id (optional) - The specific action ID to reverse
  • user (optional) - Reverse the last action for this user

/modhistory

View the moderation history for a user.

  • user (required) - The user to look up
  • page (optional) - Page number for pagination

Server Configuration

/config quarantine-role

Set the role assigned to quarantined users.

/config log-channel

Set the channel for moderation logs.

/config view

View current server configuration.

Message Management

/wipe

Delete messages from a user with optional date filtering.

  • user (required) - The user whose messages to delete
  • after (optional) - Only delete messages after this date (YYYY-MM-DD)
  • before (optional) - Only delete messages before this date (YYYY-MM-DD)

Server Rules

/rules list

Display all server rules.

/rules add

Add a new rule.

  • number (required) - Rule number
  • text (required) - Rule text

/rules remove

Remove a rule by number.

Preemptive Moderation

/preban

Ban a user ID before they join the server.

  • user_id (required) - The Discord user ID to preban
  • reason (optional) - Reason for the preban

/watchlist add

Add a user to the watchlist for monitoring.

/watchlist remove

Remove a user from the watchlist.

/watchlist view

View all users on the watchlist.

/leaveralert channel

Set the channel for alerts when watched users leave.

Bulk Role Management

/bulkrole add

Add a role to all members with a specific role.

  • target-role (required) - Role whose members will receive the new role
  • role-to-add (required) - Role to add

/bulkrole remove

Remove a role from all members with a specific role.

/bulkrole rollback

Undo a previous bulk role operation (within 24 hours).

/bulkrole history

View recent bulk role operations.

Moderation Reports

/modreport

Generate moderation activity reports.

  • period (required) - Time period: day, week, month, or all
  • format (optional) - Output format: embed, csv, or text

Duration Formats

When specifying durations, use these formats:

  • 30s - 30 seconds
  • 30m - 30 minutes
  • 2h - 2 hours
  • 7d - 7 days
  • 2w - 2 weeks
  • 1mo - 1 month
  • indefinite - No expiration

Recommended Role Setup

Option 1: Single Moderator Role

Create one role with all permissions for full access to all Bouncer Bot commands:

  • Moderate Members
  • Manage Messages
  • Ban Members
  • Manage Roles
  • Manage Server

Option 2: Tiered Moderation Team

Junior Moderator: Moderate Members only

Senior Moderator: Add Manage Messages + Ban Members

Admin: All permissions

Bot Permission Requirements

The bot itself needs these permissions:

  • Moderate Members - Apply/remove timeouts
  • Manage Roles - Quarantine system, bulk role operations
  • Ban Members - Preban functionality
  • Manage Messages - Message wiping
  • Send Messages - Respond to commands
  • Embed Links - Display rich embeds
  • Read Message History - Message wiping with date filters

Role Hierarchy

The bot's role must be positioned above any roles it needs to manage:

Setup Guide

Step 1: Invite the Bot

Contact us to get Bouncer Bot added to your server. We'll provide an invite link with the correct permissions.

Step 2: Set Up Mod Logging

Create a channel for moderation logs, then run:

/config log-channel #mod-logs

All moderation actions will now be logged to this channel.

Step 3: Set Up Quarantine Role

Create a role called "Quarantined" with minimal permissions:

  1. Create the role and place it below the bot's role
  2. Remove all channel permissions for this role
  3. Consider creating a "quarantine" channel only this role can see
  4. Run: /config quarantine-role @Quarantined

Step 4: Create Server Rules (Optional)

Define rules that can be referenced in warnings:

/rules add number:1 text:Be respectful to all members
/rules add number:2 text:No spam or self-promotion
/rules add number:3 text:Keep discussions in appropriate channels

Step 5: Set Up Leaver Alerts (Optional)

Get notified when watched users leave:

/leaveralert channel #mod-alerts

Step 6: Verify Configuration

View your current configuration:

/config view

Troubleshooting

"I cannot moderate this user"

  • The target user has a role equal to or higher than the bot
  • The target is the server owner (cannot be moderated)

"You cannot moderate this user"

  • Your role is not high enough to moderate the target user
  • You lack the required Discord permissions

Bot Can't Assign Roles

  • Move the bot's role higher in the role hierarchy
  • Ensure the target roles are below the bot's role
  • Check that the bot has Manage Roles permission
