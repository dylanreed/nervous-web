Setup Guide

Step 1: Invite the Bot

Contact us to get Plus One added to your server. We'll provide an invite link with the correct permissions.

Step 2: Set Who Can Create Invites

Define which members can create invites:

/plusone config privilege-add type:role value:<role-id>

You can add multiple rules for roles, permissions, or specific users.

Step 3: Configure Quota Mode

Choose how invite quotas are calculated:

/plusone config quota-mode mode:fixed
/plusone config quota-fixed amount:3

See Quota Modes for all options.

Step 4: Enable Trial Invites (Optional)

Let new members prove themselves with temporary access:

/plusone config trial days:7 role:@TrialMember

Step 5: Verify Configuration

Check your current settings:

/plusone status

Commands Reference

User Commands

These commands are available to users with invite privileges.

/invite create

Create a new single-use invite link.

  • trial (optional) - If true, creates a trial invite that grants temporary access

The invite link expires after 7 days if unused. Trial invites grant the configured trial role, which is automatically removed after the trial period ends.

/invite list

View your invite history and remaining quota.

Shows:

  • Current quota status
  • Pending invites (created but not yet used)
  • Used invites count
  • Expired invites count

/invite revoke

Delete an unused invite and restore your quota.

  • code (required) - The invite code to revoke

Only works on invites you created that haven't been used yet.

Admin Commands

These commands require the Manage Server permission.

/plusone config quota-mode

Set how invite quotas are calculated.

  • mode (required) - One of: fixed, tiered, replenishing, earned

/plusone config quota-fixed

Set the default invite quota (used in fixed mode and as fallback).

  • amount (required) - Number of invites (0-100)

/plusone config quota-tier

Add or update a role-based quota tier.

  • role (required) - The role to configure
  • amount (required) - Number of invites for this role
  • priority (optional) - Higher priority wins if user has multiple roles (default: 0)

Example:

/plusone config quota-tier role:@Moderator amount:10 priority:5
/plusone config quota-tier role:@Member amount:3 priority:1

/plusone config quota-replenish

Configure automatic quota restoration (for replenishing mode).

  • amount (required) - Invites to restore per period
  • days (required) - Days between restorations

Example: Grant 1 invite every 30 days:

/plusone config quota-replenish amount:1 days:30

/plusone config trial

Configure trial invites.

  • days (required) - Trial duration in days (use 0 to disable)
  • role (required when enabling) - Role to grant during trial

When trial expires, the role is removed but the user stays in the server.

/plusone config privilege-add

Define who can create invites.

  • type (required) - Rule type: role, permission, or user
  • value (required) - Role ID, permission name, or user ID
  • quota (optional) - Override quota for this rule

Valid permission values:

  • ADMINISTRATOR
  • MANAGE_GUILD
  • MANAGE_ROLES
  • KICK_MEMBERS
  • BAN_MEMBERS
  • MODERATE_MEMBERS

If no rules are configured, defaults to allowing users with Manage Server permission.

/plusone config privilege-remove

Remove a privilege rule.

  • rule_id (required) - The rule ID (shown in /plusone status)

/plusone grant

Manually grant invite quota to a user (earned mode only).

  • user (required) - User to grant quota to
  • amount (required) - Number of invites to grant

/plusone status

Display current bot configuration.

Shows:

  • Quota mode and default quota
  • Replenish rate (if applicable)
  • Trial invite settings
  • Quota tiers
  • Privilege rules

/plusone audit

View invite usage history.

  • user (optional) - Filter by invite creator

Shows total invites created, used, pending, and expired.

Permissions Guide

Bot Permissions

Plus One requires these Discord permissions:

Permission Why It's Needed
Manage Server Read server invites to track which invite was used when members join
Manage Roles Add and remove trial roles from members
Create Instant Invite Generate invite links on behalf of users

Privileged Intents

The bot requires Server Members Intent to detect when members join/leave.

Role Hierarchy

For trial roles to work, the bot's highest role must be above the trial role:

Bot Role        ← Must be here or higher
Trial Role      ← The role granted to trial members
@everyone

User Permissions

Commands Requirement
All /plusone commands Manage Server permission
/invite commands Configurable via privilege rules

Troubleshooting

"Cannot create trial invites"

The bot cannot manage the trial role. Check:

  • Bot has Manage Roles permission
  • Bot's highest role is above the trial role

"Failed to create invite"

The bot cannot create invites in this channel. Check:

  • Bot has Create Instant Invite permission
  • Channel permissions don't override this

Invite tracking not working

The bot cannot read server invites. Check:

  • Bot has Manage Server permission
  • Privileged intents are enabled

Quota Modes

Choose how invite quotas are calculated for your server.

Fixed Mode

Everyone gets the same number of invites.

/plusone config quota-mode mode:fixed
/plusone config quota-fixed amount:5

Best for: Simple communities where all members are treated equally.

Tiered Mode

Different roles get different quota amounts.

/plusone config quota-mode mode:tiered
/plusone config quota-tier role:@Moderator amount:10 priority:5
/plusone config quota-tier role:@Trusted amount:5 priority:3
/plusone config quota-tier role:@Member amount:2 priority:1

Best for: Communities that want to reward trusted members with more invite power.

Note: If a user has multiple roles, the highest priority tier is used.

Replenishing Mode

Quotas restore automatically over time.

/plusone config quota-mode mode:replenishing
/plusone config quota-fixed amount:3
/plusone config quota-replenish amount:1 days:30

This grants 1 invite every 30 days, up to the maximum of 3.

Best for: Slow, sustainable growth over time.

Earned Mode

Admins manually grant invite quota as rewards.

/plusone config quota-mode mode:earned
/plusone grant user:@someone amount:5

Best for: Exclusive communities where invites are special rewards.
